Introdução:

  • Pronunciar as CSS ou a CSS é indiferente, pois ambas as formas de expressão estão corretas;
  • CSS (Castating Style Sheeds) signifa Folhas de Estilo em Cascata;
  • As CSS podem ser aplicadas de 3 (três) formas:
    1. In Line: é o mais fraco de todos e menos recomendado. Mas é um bom começo, e implica em dizer que o ESTILO será aplicado na mesma linha que a SEMÂNTICA;
    2. CSS Interno ou CSS Local: nessa técnica o estilo é aplicado de forma separado, facilitando a visualização e permitindo com maior facilidade, por parte do desenvolvedor, em identificar o que é SEMÂNTICA ou ESTILO;

  • Vale lembrar que o HTML5 foi projetado para dar ênfase a SEMÂNTICA, ou seja, dar sentido ao que foi escrito para a internet. Então a diferença no tamanho das letras implica em dizer que o desenvolvedor quiz dar um significado aquela ação de escrever o texto como letras de tamanho diferente;
  • Essa técnica de aplicação de estílo tem uma GRANDE INCOVENIÊNCIA que é o fato de ter que realizar as aplicações localmente, então, o código contiver 200 aplicações de estilo diferentes será bem trabalhoso alterar cada uma individualmente. Por esse motivo que a técnica In Line é a mais fraca de todas;
  • A técnica In Line também apresenta outra desvantagem que é a POLUÍÇÃO do código, deixando a MANUTENÇÃO complicada para o desenvolvedor. Fica difícil definir o que é SEMÂNTICA e o que é ESTÍLO;

  • O Método In Line apesar de ser atual e moderno não é reconmendável para o DIA A DIA e sim para situações pontuais onde as outras técnicas se mostraram não muito eficiente.

Estilo: Locais ou Internos

  • Nessa forma de aplicação de estilo serão colocados seletores na parte HEAD do códido que vão corresponder aos locais de aplicação do estilo;
  • Os seletores estarão dentro da TAG <style> acompanhadas por CHAVES. Então, se o estilo tiver de ser aplicado a tag <body> coloca-se dentro da TAG <style> acompanhada por chaves, o mesmo procedimento deve ser seguido para outras TAGs.
  • Click no link para que o resultado possa ser observado.
        <!DOCTYPE html>
        <html lang="pt-br">
        <head>
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <link rel="shortcut icon" href="favicon.ico" type="image/x-icon">
            <title>CSS Local ou Interno</title>
            <style>
                body {
                background-color: lightblue;
                font-family: Arial, Helvetica, sans-serif;
                font-size: 20px;
                }
                h1 {
                color: darkblue;
                background-color: darkcyan;
                }
                h2 {
                color: darkred;
                }
                p {
                text-align: justify;
                }
            </style>
        </head>

  • Como foi percebido basta colocar somente um seletor para que todos tenham o mesmo efeito, então, se existirem 200 tags <h1> só será preciso aplicar o estilo somente 1 vez;
  • Cada linha da TAG <style> dentro de HEAD chama-se DECLARAÇÃO que se resume por um conjunto de PROPRIEDADE e VALOR;
  • Com essa técnica pode-se perceber que o código ficou muito melhor apresentado a SEMÂNTICA está separada do ESTÍLO;
  • Essa técnica é recomendada quando o desenvolvedor tiver uma página construída e um estílo deve ser aplicada a ela sem que outras páginas seja afetadas
  • Tags antigas devem ser evitadas;
  • A técnica INTERNA de aplicação de estilo apresenta 2 desvantagens:
    1. A tendência das configurações de ESTILO se tornarem miores que o conteúdo do SITE. Isso acarreta em um incômodo no momento em que se desejar abrir o código para manutenção ou ajuste;
    2. Se o desenvolvedor criar cerca de 30 páginas com o mesmo estilo. A parte do códito de configuração terá que ser copiada para as outras também.
